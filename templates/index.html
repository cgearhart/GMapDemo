This is a proof of concept visualization of a RESTful health monitoring API for the Tesla SuperCharger system, written in response to the job posting at Tesla Careers. The API is run on Django (powered by django-rest-framework & django-registration for user management) and hosted on Heroku running a PostGreSQL database.The visualization relies on a small javascript file, hosted on Amazon s3 (along with static files for the other libraries), to plot the location of SuperCharger stations, color coded for simulated health condition (green for 'Operational', blue for 'Scheduled Maintenance', yellow for 'Unscheduled Maintenance', and red for 'Offline'). It took a few hours to implement and debug. With appropriate account credentials, the API is actually already functional; the database schema is not well-defined for complete health monitoring, but it demonstrates the intended functionality. There are endpoints for User Accounts, Admin Control, POSTing/GETting supercharger stations, and POSTing/GETting events on those stations - with optional filtering on stations and events.
